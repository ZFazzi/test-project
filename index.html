These are some things I've learned about Git and Github

    - Git Bash is where your work is localy stored.
    - You can push work you've done to GitHub where your teamates can access the code I've edited in a branch outside the master code.
    - You can also pull the most updated version of the master, dont forget to do this before you push your code.
    - In Git you can also create branches and choose which branch to make requests to.
    - When you push a branch, i.e post it to Github, its a sort of draft that needs to get reviewed by the team manager before being merged into the master code.
    - When a branch is merged it is submitted into the master code and no longer a draft.
    - Multiple branches can be created to the master code at the same time, it can be compared to a tree, where all the branches come from the same roots.
